◆ソフトの概要

	ZipPla をご利用頂きありがとうございます。
 	ZipPla は、圧縮ファイルや PDF 等として保存された書籍データの検索・閲覧を目的に開発がスタートしました。
	今ではあらゆるファイルをレート・タグで管理できる他、動画・テキスト等もサムネイル表示可能です。

	主な特徴
	・大量のファイルをレート・タグ機能や検索機能等で快適に管理
	・単体で zip, jpeg 等多様なファイル形式をサムネイル表示する他、ffmpeg により動画ファイルにも対応
	・柔軟な外部プログラム呼び出し
	・独自の画像解析により画像内の適切な部分を自動的にトリミングしてサムネイル表示


◆動作環境

	Windows 10 Pro 64-bit で動作確認しています。
	Windows 10/8.1/7/Vista （それぞれ 32-bit/64-bit 版）での動作不良は不具合として対応致します。


◆対応ファイル形式

	任意のファイルを管理することができますが、以下のファイルはサムネイルの表示・内蔵ビューアによる閲覧も可能です。

	圧縮ファイル／PDF ファイル： zip/rar/7z/cab/tar/cbz/cbr/epub/pdf/rar5/lzh（※１，※２）
	画像ファイル： bmp/gif/jpeg/png/apng/wdp/hdf/jxr/exf/tiff/ico/emf/wmf/dds/tga/psd/jp2/webp（※３，※４）

	以下のファイルのサムネイルも表示できます。
	
	文章ファイル： txt/ini/html
	音声ファイル： mp3/wav/m4a/ogg/ape/flac 他、全 59 種類
	動画ファイル： mp4/wmv/mov/avi/mpeg/qt/asf/3gp/3g2/rmvb/f4v/ts/mts/m2ts/m2v/m4v/vob/mkv/ogv/ogm/flv/webm/divx（※５）

	※１ 画像を含むフォルダも圧縮ファイルと同様に扱うことができます。
	※２ 一部のマルチボリューム書庫と rar5/lzh の表示には 7z.dll (https://sevenzip.osdn.jp/) が必要です。
	　　 7z.dll は 7-Zip をインストールするか手動で ZipPla.exe と同じ場所に配置することで導入できます。
	　　 7z-x86.dll / 7z-x64.dll という名前に変更し、環境に応じて自動的に dll を選択させることもできます。
	※３ 内蔵ビューアはアニメーション GIF (gif) 及びアニメーション PNG (png/apng)に対応します。
	※４ jp2/webp の表示には ffmpeg.exe が必要です。詳しくはプログラム起動時のメッセージをご覧下さい。
	※５ 動画のサムネイル表示には ffmpeg.exe が必要です。詳しくはプログラム起動時のメッセージをご覧下さい。


◆インストール・アンインストール方法

	ZipPla はレジストリを使用しません。ダウンロードした ZIP ファイルを任意のフォルダに解凍して下さい。
	アンインストール時はフォルダごと削除すれば OK です。


◆使用方法

	プログラム起動後、ドラッグアンドドロップやメニューコマンドから管理したいファイルのあるフォルダを指定して下さい。
	詳しくは ZipPla のホームページをご覧下さい。
	https://sites.google.com/site/riostoolbox/zippla

	・サムネイルが表示されるまでの間ファイルアイコンを表示するようにした
	・サムネイル上のレート領域に表示されるポップアップが時間経過で自動的に消えるようにした

	・内蔵ビューアでズームした際に画像の辺が描画領域の辺の長さを超えなかった場合のマウスカーソルの挙動を改善

	・内蔵ビューアーで、マウスジェスチャ有効時にも右クリックとその他のキーを組み合わせた入力を
	　受け付けられるようにした
	・内蔵ビューアーで、A と　A+B の両方のキー入力にコマンドが割り当てられているとき
	　「A押す → B押す → B離す → A離す」というキー入力で A のコマンドが実行されないようにした
	（詳細な仕様の説明：
	　ZipPla では、ユーザーが設定したショートカットキー同士の競合状態に応じて
	　コマンドが実行されるタイミングがキーを押した瞬間か離した瞬間か自動的に選択されます。
	　従来の動作は、「A押す → B押す → B離す → B押す」という入力が行われたときに A+B の
	　コマンドが２回実行されるという動作を実現するため、「A押す」後のプログラムの内部状態と
	　「A押す → B押す → B離す」後の内部状態とを同じにしたことが原因で起こっていました。
	　今回の変更では、一連のキー入力の中で一度でもコマンドが実行されると
	　それ以降はキーを離した瞬間に実行されるコマンドを無視するようにしています。
	　従って A+B がキーを離した瞬間に実行されるコマンドであるときに
	　「A押す → B押す → B離す → B押す → B離す」という入力を行うと、従来は A+B のコマンドが
	　２回実行されていましたが今回のバージョンからは１回しか実行されなくなります。）
